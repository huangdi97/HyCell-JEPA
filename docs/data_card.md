# Data Card

## Dataset Status
The project currently uses a small deterministic toy single-cell perturbation dataset generated by `scripts/make_toy_data.py`. Real data integration scaffolding now exists for validating candidate `.csv`, `.npz`, and `.h5ad` files, but no real dataset is included or downloaded automatically.

## Toy Data Purpose
Toy data is for engineering validation only. It should test data loading, gene set scoring, encoders, transition modeling, benchmarking, and demo flows.

## Planned Toy Data Contents
- HDF-like toy cells.
- Small marker gene panel.
- Perturbation labels for control, aging-like stress, regeneration-like action, and partial reprogramming-like action.
- Timepoint metadata.
- Compact readout scores from configured gene sets.

## Current Toy Files
- Config: `configs/toy_data.yaml`
- Generator: `scripts/make_toy_data.py`
- Default generated cells: `outputs/toy_data/toy_cells.csv`
- Gene set config: `configs/gene_sets.yaml`
- Scoring script: `scripts/score_gene_sets.py`
- Default generated scores: `outputs/toy_data/gene_set_scores.csv`
- Evidence graph: `outputs/toy_data/evidence_graph.json`

## Real Data Plan
Current real-data scaffolding:
- Config: `configs/real_data_schema.yaml`
- CLI validator: `scripts/validate_dataset.py`
- Loader interfaces: `src/hycell/data_loaders.py`
- AnnData-like schema validation: `src/hycell/schema.py`
- Perturbation and HDF aging metadata adapters: `src/hycell/perturbation_adapter.py`

Supported interfaces:
- `.csv`: loads metadata columns plus numeric gene-expression columns.
- `.npz`: loads `expression` or `X`, gene names, and per-cell metadata arrays.
- `.h5ad`: uses optional `anndata` if installed; otherwise fails with a clear dependency message.

Future goals should connect validated real data into scoring, adapters, benchmarks, and model evaluation.

## Restrictions
- Do not download large datasets automatically.
- Do not commit large data files.
- Clearly mark toy data as synthetic or toy.
- Do not claim toy-data behavior is biological evidence.
- Treat real-data candidate validation as schema checking, not biological interpretation.
