# Data Card

## Dataset Status

HyCell-JEPA v0.1 uses one toy dataset and one real public single-cell matrix smoke workflow.

No raw real data, processed real-data NPZ files, generated outputs, checkpoints, embeddings, `.h5ad`, `.npy`, `.npz`, or `.zip` artifacts should be committed.

## Toy Data

Toy data is generated by:

```bash
python scripts/make_toy_data.py --config configs/toy_data.yaml
```

Purpose:
- Exercise file formats.
- Test gene-set scoring.
- Train compact toy encoders and JEPA transition core.
- Run verifier, planner, benchmark, and demo workflows.

Toy data includes HDF-like synthetic cells, small marker panels, toy perturbation labels, toy timepoints, and compact readout scores.

Toy data is engineering validation only and is not biological evidence.

## GSE130973 Raw Files

GSE130973 is the first real public skin aging single-cell smoke dataset used by the project.

Expected manually downloaded processed GEO files:

```text
data/raw/gse130973/GSE130973_barcodes_filtered.tsv.gz
data/raw/gse130973/GSE130973_genes_filtered.tsv.gz
data/raw/gse130973/GSE130973_matrix_filtered.mtx.gz
```

These files are ignored by git and are not redistributed by this repository.

## GSE130973 Processed Smoke NPZ

The processed smoke artifact is created by:

```bash
python scripts/prepare_gse130973.py --raw-dir data/raw/gse130973 --out data/processed/gse130973/gse130973_smoke.npz --max-cells 5000 --max-genes 2000
```

Expected contents:
- `expression`
- `gene_names`
- `cell_ids`
- `perturbation`
- `timepoint`
- `cell_system`
- `state_label`
- `age_label`
- `dataset_id`
- `source`
- `is_real_data`
- `note`

The default smoke artifact is capped at 5000 cells x 2000 genes and uses cells x genes orientation.

## Why Age Labels Are Unknown

The three processed GEO matrix files provide expression values, genes, and barcodes. They do not provide reliable donor age or sample-level metadata in the files used by the smoke workflow. Therefore `age_label` and `state_label` are set to `unknown`.

No age labels are inferred or invented.

## Why The Current Smoke File Is Not Fibroblast-Only

The three processed GEO matrix files do not provide enough cell-type annotation to claim a fibroblast-only or HDF-only subset. The project therefore uses:

```text
cell_system = skin_single_cell_unfiltered
```

Downstream HDF/fibroblast filtering requires reliable metadata or cell-type annotation.

## Future Metadata Plan

Planned data improvements:

- Add reliable GSE130973 sample metadata if available.
- Add cell-type annotations or a documented annotation workflow.
- Create an explicit HDF/fibroblast subset only after metadata supports it.
- Add small scPerturb and perturbation benchmark datasets without automatic large downloads.

## Data Safety

- Do not download large datasets automatically.
- Do not commit `data/raw/`.
- Do not commit `data/processed/`.
- Do not commit `outputs/`.
- Do not commit generated `.npz`, `.npy`, `.h5ad`, `.pt`, `.ckpt`, or `.zip` files.
- Treat all toy and smoke outputs as engineering artifacts.
