# Model Card

## Model Purpose

HyCell-JEPA v0.1 is an engineering prototype for compact cellular world modeling. It tests whether a small, inspectable JEPA-style transition stack can predict compact toy biological belief states and support verifier/planner plumbing before attempting real biological claims.

## Inputs

Toy transition workflow:
- Compact gene-set readout state `b_t`.
- Toy action label `a_t`.
- Context label `c_t`.
- Cell-system adapter label `h_t`.

Real-data smoke workflow:
- Processed GSE130973 NPZ with `expression`, `gene_names`, `cell_ids`, dataset metadata, and unknown state/age labels.

## Outputs

Toy workflow:
- Predicted compact next-state readouts.
- Encoder checkpoints and embeddings.
- JEPA metrics and benchmark reports.
- Verifier outputs and toy planner sequences.

Real-data smoke workflow:
- Matrix summary JSON/Markdown.
- Encoder-style real-matrix embeddings.
- Real smoke metrics and report.

## Training Objective

The toy JEPA core uses a deterministic ridge-regression transition objective over compact toy readouts. The objective is software validation of the transition interface, not biological discovery.

The GSE130973 real-data training smoke uses a deterministic projection after normalization. It is an encoder-style matrix smoke test, not a perturbation-transition model.

## Current Datasets

- Deterministic toy HDF-like perturbation data generated by `scripts/make_toy_data.py`.
- GSE130973 processed GEO files, manually downloaded by the user and converted into `data/processed/gse130973/gse130973_smoke.npz`.

The repository does not commit raw real data, processed real-data NPZ files, checkpoints, embeddings, or generated outputs.

## Intended Use

- AI4Science engineering portfolio review.
- Local reproducibility demos.
- Testing data-loader, schema, encoder, verifier, planner, and release automation patterns.
- Scaffolding future real-data HDF/fibroblast modeling after metadata annotation is available.

## Out-Of-Scope Use

- Clinical, diagnostic, therapeutic, or dosing recommendations.
- Wet-lab decision making.
- Biological discovery claims from toy or smoke metrics.
- Full virtual-cell simulation.
- Full transcriptome generation or Lingshu-Cell-scale diffusion.
- Planner recommendations for interventions.

## Risks And Limitations

- Toy data is synthetic engineering validation only.
- GSE130973 smoke output is unfiltered human skin single-cell data, not HDF-only.
- Age and state labels are unknown from the three processed GEO matrix files alone.
- No perturbation transitions are learned from GSE130973.
- Real-data smoke embeddings are not biological representations validated against cell type, donor age, intervention, or outcome.
- The verifier is a software guardrail, not a biological safety system.

## Reproducibility Notes

Use:

```bash
pytest
bash scripts/verify_release.sh
```

Real GSE130973 workflows require manually placing processed GEO files under `data/raw/gse130973/`. No dataset download is performed by tests or verifiers.
